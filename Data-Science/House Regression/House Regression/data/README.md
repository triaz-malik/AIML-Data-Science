## Schema highlights

The dataset describes residential homes in Ames, Iowa. 80 explanatory variables span:

- **Size** — `GrLivArea`, `TotalBsmtSF`, `1stFlrSF`, `GarageArea`, `LotArea`
- **Quality / condition** — `OverallQual`, `OverallCond`, `KitchenQual`, `BsmtQual`, ... (ordinal Po/Fa/TA/Gd/Ex)
- **Location** — `Neighborhood` (25 levels), `Condition1/2`, `MSZoning`
- **Time** — `YearBuilt`, `YearRemodAdd`, `YrSold`, `MoSold`
- **Type** — `BldgType`, `HouseStyle`, `RoofStyle`, `Foundation`, `SaleType`

Target: `SalePrice` (USD). Strongly right-skewed (skew ≈ 1.88) — the pipeline predicts `log1p(SalePrice)` to align with the RMSLE evaluation metric.

## Missing-value semantics

Most NaN entries encode **feature absence**, not unknown data — see `raw/data_description.txt`. The pipeline fills them accordingly:

| Strategy | Examples | Rationale |
|---|---|---|
| Literal `"None"` | `PoolQC`, `Alley`, `Fence`, `FireplaceQu`, `Garage*`, `Bsmt*` | NaN means "no pool / alley / ..." |
| `0` | `GarageYrBlt`, `MasVnrArea`, `BsmtFinSF*`, `TotalBsmtSF` | Absent feature → zero size |
| Neighborhood median | `LotFrontage` | Truly missing; street-level median is best proxy |
| Mode | `MSZoning`, `Electrical`, `KitchenQual`, ... | Single-NaN columns |
